Fleet CP1252 Mojibake Fix

Unicode glyph → ASCII equivalent · runtime output only · classify before touching

Purpose
When shell scripts (bash/PowerShell) print non-ASCII glyphs like checkmarks, X marks, or box-drawing rules, they render as mojibake under Git Bash / Windows cp1252 terminals. Replace runtime output with ASCII equivalents while leaving comments and remote heredocs untouched.
Auditing a script fleet
Console output across bash/PowerShell needs cross-terminal safety review.
Migration pass
Sweeping helper functions for glyph decoration during a refactor.
Enforcement going forward
Pre-commit hook needed to block ASCII violations before they land.
six-step method, classify → detect → replace → verify
Method
  • 1
    Classify every non-ASCII byte first
    Only a problem in runtime-emitted output. Fine in: source comments, remote SSH heredocs, data string literals.
  • 2
    Detect offending bytes
    cat -A <file> or perl -ne 'print if /[^\x00-\x7F]/'; grep for the specific glyphs to find offending helper functions.
  • 3
    Replace runtime glyphs with ASCII
    See glyph substitution table below.
  • 4
    Remove offending blocks by line-range
    awk line-range deletion — content-agnostic, won't strip adjacent comments or unrelated code.
  • 5
    Verify with a residual-byte scan
    Re-run the non-ASCII scan; confirm every remaining byte is one of the three acceptable categories from step 1.
  • 6
    Enforce going forward
    Pre-commit hook blocks non-ASCII runtime output for .ps1. .sh fleet: fix manually during refactoring passes — no reliable static enforcement point exists.

Gotchas
  • Don't blanket-strip all non-ASCII — comments and remote heredocs are legitimate; stripping them is unnecessary churn and can break UTF-8-aware remote content.
  • awk line-range deletion is preferred over pattern-based deletion for whole helper blocks — avoids orphaned braces/parens.
  • The bug only manifests under cp1252 (Windows Git Bash, cmd.exe) — the same script may look fine tested from a UTF-8 Linux terminal.
  • Always re-verify after cleanup — a partial glyph replacement (fixing ✓ but missing ══ in the same helper) is a common miss caught only by the residual-byte scan.
