Shell Helper Migration

consolidate log/ok/fail/die into one sourced module · preserve exit-code semantics exactly

When to use
Consolidating duplicated shell helper functions (log, ok, fail, die, section, info, warn, colour variables) scattered across many bash scripts into one shared module — without breaking exit codes, captured output, or execution semantics. Applies to batch migrations across tens or hundreds of scripts.
identify → source one line → forensically read contracts → judge per-file → clean → verify → batch
Method
  • 1
    Identify by canonical name
    Delete only standard helper names; keep custom ones (ssh_devops, hr, kv) untouched.
  • 2
    Add exactly one source line
    Immediately after set -euo pipefail. Anchor on SCRIPT_DIR if present, else $(dirname "${'$'}{BASH_SOURCE[0]}") as the safe default.
  • 3
    Forensically determine each helper's contract
    Does the def exit 1? return N? Is the call site followed by explicit exit/die, or bare inside a loop? Is set -e active?
  • 4
    Judge per-file on three axes
    Execution context (repo checkout vs. remote/on-host — see skip-taxonomy skill), contract (non-standard exit codes), semantics (fatal vs. non-fatal call sites).
  • 5
    Clean up orphaned state
    Grep for now-unused colour vars ($GREEN, $RED, _ts) and remove — only if nothing else in the file still uses them.
  • 6
    Guard against use-before-source
    bash -n only checks syntax, not order. Grep for helper calls above the source line (common in arg-parsing loops); move the source line earlier if found.
  • 7
    Verify
    bash -n on every changed file, no-color render preview, smoke-test the arg-parse path (--bogus should fail via sourced fail()).
  • 8
    Batch coherently
    Group by filename prefix / subsystem, not the whole repo in one commit — keeps review and revert tractable across 100+ files.
Exit-code decision rule
fail + exit N (N≠1)stays exactly as-is
fail + exit 1, or ... || diebecomes die
bare non-fatal fail (fleet loop)stays fail
Gotchas
  • A return-based fail() under set -e aborts immediately — fail; fail; exit 1 never prints the second message. Flag it, don't silently preserve.
  • Never blind-sed replace fail→die or vice versa; read semantics from def and every call site.
  • Removing non-ASCII glyphs is fine workstation-side, but NOT inside heredoc payloads — see skip-taxonomy skill.
  • "Looks the same" is not "is the same" — identically-named helpers can carry different exit-code contracts.
