Output Contract · applies to every operation
The user wants signal, not narration. Terminal output for any operation is exactly these three things and nothing else. This overrides any conversational or explanatory default for the duration of the run.
1 · Stay silent
During execution, run the commands — no "Let me check…", no per-command play-by-play, no preamble.
2 · Errors as they occur
When a command fails, print the failing command + stderr verbatim, then stop or ask. Errors are the only mid-run interruption.
3 · One concise summary
End with a single ≤4-line block: what landed, where (PR # · SHA · tag · branch), and any caveat to act on.
seven thin commands + one direct-trigger sub-skill, one lifecycle
Operations · commands/ → sub-skills/
  • /publish
    Publish a hardened repo (day 0)
    gh repo create + gitleaks pre-commit hook + .gitignore/.gitattributes + main / dev with branch protection requiring PRs on both.
  • /commit
    Stage · pull · commit · push
    Conventional-commit message from the staged diff. Warns on sensitive files; never force-pushes.
  • /ship
    Feature branch → PR → dev
    Branch preflight (create/rename to type/slug), generic test gate, optional /code-review, open PR, merge commit, clean up.
  • /merge
    Merge open PR(s) into dev
    Targets = PR # · branch · worktree · a set · or current context. Gates on mergeable + CI, merges, cleans up worktree → local → remote.
  • /release
    dev → main versioned release
    Branch preflight, auto semver from conventional commits, --no-ff merge, tag + GitHub Release, sync dev.
  • /release-init
    Provision / repair release automation
    Release-Automation Standard: persistent changelog generator + tag-triggered release.yml that regenerates notes from git at tag time — a committed CHANGELOG is a cache, never the source of truth. Detects missing / anti-pattern / stale states; idempotent.
  • /prune
    Prune stale worktrees + branches
    Merged-into-dev detection (squash or standard), protects dirty worktrees, deletes local + origin, logs to docs/git-log.md.
  • no cmd
    worktree-task-lifecycle · triggers directly
    Single lifecycle authority: create under <repo>-wt/.worktrees/, clean-baseline setup, credentials stay in the primary checkout, merge from primary (never inside the worktree), post-merge idempotent removal with Windows lock recovery. merge/prune and agent harnesses delegate here.

Parameter Contract · /merge targets
Optional message resolves to…
1209 = PR number · a branch name · a worktree path · a set of any of these · or empty → current context (current branch's open PR, or the worktree you stand in).
Missing target
When nothing resolves, elicit it via AskUserQuestion — never inline-print a free-text question.
Cross-Operation Principles
  • Protected branches — never push to or delete dev/main.
  • Merge strategy is a merge commit; never squash feature → dev.
  • Run destructive git/gh from the repo root, never a secondary worktree.
  • Cleanup order: worktree → local branch → remote.
  • Worktree create/remove goes through worktree-task-lifecycle (incl. Windows lock recovery).
  • Commit conventions — type: description, atomic commits, no WIP in PRs.
  • Verify remembered refs live (git ls-remote) before anything destructive; repair with git remote set-head origin -a.
  • Never bypass a failing CI gate — surface and stop.
