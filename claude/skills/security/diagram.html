Defense layers

01Secrets & Environment

  • Non-negotiable .gitignore set
  • .env.example, validated at startup (Zod / Pydantic)
  • Never VITE_* / NEXT_PUBLIC_* for secrets

02Input Validation

  • Parameterized queries only (SQLi)
  • Sanitized HTML with allowlists (XSS)
  • File uploads: size, type, extension
  • Path traversal guards

03AuthN & AuthZ

  • Short-lived JWTs, httpOnly cookies
  • Authorization before sensitive ops
  • Supabase Row Level Security
  • bcrypt 12+ rounds · rate limiting

04CSRF & Headers

  • CSRF tokens on state changes
  • SameSite=Strict cookies
  • helmet / CSP via next.config.js

05Data Exposure

  • Redacted logging — no PII, no tokens
  • Generic error messages to users
  • Stack traces stay server-side

06Automation & Deps

  • detect-secrets pre-commit · security-check.sh
  • GitHub Actions: trufflehog, audit, CodeQL
  • npm audit / safety · lock files committed
  • Solana wallet & transaction verification
