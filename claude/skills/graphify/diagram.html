Graphify

any input → knowledge graph → clustered communities → HTML + JSON + audit report

What graphify does that assistant memory alone cannot
1 · Persistent graph
Relationships stored in graphify-out/graph.json survive across sessions — ask weeks later without re-reading.
2 · Honest audit trail
Every edge tagged EXTRACTED, INFERRED, or AMBIGUOUS — you know what was found vs invented.
3 · Cross-document surprise
Community detection finds connections between concepts across files you'd never think to ask about directly.
nine-step pipeline · /graphify <path>
Pipeline
  • 1
    Ensure graphify is installed
    Detect Python via uv → pipx → active env; install if missing.
  • 2
    Detect files
    Classify corpus into code/docs/papers/images/video. Warn + ask for a subfolder if >200 files or >2M words.
  • 2.5
    Transcribe video/audio (conditional)
    Skip if zero video files. Otherwise Whisper-transcribe with a domain-hint prompt written from god-node labels.
  • 3
    Extract entities + relationships
    Part A (AST, code, deterministic) and Part B (semantic, parallel subagents) run simultaneously. Part C merges both.
  • 4
    Build · cluster · analyze · generate
    networkx graph → community detection → cohesion scoring → god nodes → surprising connections → GRAPH_REPORT.md + graph.json.
  • 5
    Label communities
    2-5 word plain-language names per community; regenerate report + questions with real labels.
  • 6
    Generate outputs
    HTML always (unless --no-viz); Obsidian vault + canvas only if --obsidian.
  • 7
    Optional exports
    --neo4j / --neo4j-push / --svg / --graphml / --mcp stdio server.
  • 8
    Token reduction benchmark
    Only if corpus > 5,000 words — otherwise the graph's value is structural clarity, not compression.
  • 9
    Manifest · cost tracker · cleanup · report
    Save manifest for --update, log cumulative token cost, delete temp files, print outputs + top report sections.
Decision points · mode flags
--update
Incremental — re-extracts only new/changed files; skips semantic pass entirely if all changes are code-only.
--cluster-only
Skips extraction, re-runs clustering on the existing graph.
--mode deep
Aggressive INFERRED edges — indirect deps, shared assumptions, latent couplings.
--watch
Background watcher; code changes auto-rebuild, doc/image changes flag needs_update.
--directed
Preserves edge direction source→target instead of undirected graph.
--mcp
Starts stdio MCP server exposing query_graph, get_node, shortest_path, god_nodes, etc.
Query modes · /graphify query
ModeFlagBest for
BFS (default)(none)"What is X connected to?" — broad context, nearest neighbors first.
DFS--dfs"How does X reach Y?" — trace a specific chain or dependency path.
Also: /graphify path A B (shortest path) and /graphify explain "X" (plain-language node explanation). Every answer is written back into the graph via save-result — closes the feedback loop for the next --update.
Companion flows
/graphify add <url>
Fetch (Twitter/X, arXiv, PDF, image, or generic webpage), save to ./raw, auto-run --update.
Git commit hook
graphify hook install — re-extracts changed code files and rebuilds after every commit.
CLAUDE.md integration
graphify claude install — makes graphify always-on for future Claude Code sessions in this project.
Neo4j push
--neo4j-push bolt://... uses MERGE — safe to re-run without duplicating nodes.
Honesty rules
  • Never invent an edge — if unsure, mark it AMBIGUOUS.
  • Never skip the corpus-size warning at >200 files / >2M words.
  • Always show token cost in the report.
  • Never hide cohesion scores behind symbols — show the raw number.
  • Never run HTML viz on a graph with more than 5,000 nodes without warning the user.
