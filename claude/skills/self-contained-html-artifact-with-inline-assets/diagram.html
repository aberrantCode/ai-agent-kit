Self-Contained HTML Artifact

zero external requests · inline everything · light & dark theme correctness

When to use
A deliverable needs to be portable enough to email, copy to a USB drive, or open as a bare file — a design-review mockup, static dashboard, slide deck, data browser — and must work with no external fetches, no relative paths, no server. Also applies when a strict CSP blocks all external hosts.
8-step method, inline-first
Method
  • 1
    Inline everything
    CSS, SVG icons, images all inside the single HTML file. Strip @import. Embed as data URIs or shrunk rasters if a data URI would bloat the file.
  • 2
    CSS gradients as placeholder "photos"
    Unblocks design review without waiting on an asset pipeline.
  • 3
    Inline favicons and touch icons
    Declare directly in <head>; don't reference /static/ paths unless tested end-to-end. Verify legibility at 16px and 32px.
  • 4
    Author SVGs fully self-contained
    Style, color, dimensions hand-inlined in the SVG itself — no external stylesheet refs, no CSS custom properties, no context-dependent data URIs.
  • 5
    Responsive scaling via container queries
    Use cqi units, not viewport units — scales correctly regardless of embedding container.
  • 6
    Build both theme paths
    Light and dark, not just a default. Screenshot with theme pinned and transitions disabled to avoid capturing a mid-crossfade frame.
  • 7
    Generator script for larger artifacts
    Assemble the final single-file output from modular, readable sources at build time; keep sources editable, dist/ regenerable.
  • 8
    Verify before shipping
    Open in a real browser, confirm dynamic behavior, toggle theme, confirm zero network-tab requests.
House constraints
No CDNs, no build step
Must open directly as a bare file — no relative-path assets, no server dependency.
CSP-safe
Works even when a strict CSP blocks fonts, scripts, images from external hosts.
Gotchas
  • A data-URI SVG in page CSS-variable context looks self-contained but isn't reusable — hand-author it independent of the page.
  • Screenshotting mid-CSS-transition captures a blended, wrong frame — disable transitions first.
  • "Opens in a browser" ≠ "zero external requests" — check the network tab, not just visual rendering.
  • Don't let the generated dist/ artifact become source of truth — always edit modular sources and regenerate.
