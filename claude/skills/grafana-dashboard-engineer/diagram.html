Grafana Dashboard Engineer

research → design → build → deploy → validate → ship · Prometheus/Loki · IaC-integrated

Purpose
Production-grade Grafana dashboard engineer. Enables rapid research, design, build, deployment, and validation of observability dashboards across Prometheus, Loki, and custom datasources — with IaC (Ansible/PowerShell), application-specific templates, and full git workflow integration for homelab and enterprise environments.
Domain expertise
Dashboard JSON, PromQL/LogQL, app-specific templates, Ansible/PowerShell IaC, SLO/SLA tracking.
Workflow integration
AskUserQuestion requirements gathering, feature-branch worktree, live API sync, ship-to-dev.
Application templates
Searchable library of pre-built monitoring patterns, auto-researched when missing.
five slash commands, one dashboard lifecycle
Slash Commands
  • /new-dashboard
    Create from scratch, interactive
    Detect structure → find/research template → AskUserQuestion requirements → build JSON → test queries → generate docs → offer live deploy.
  • /update-dashboard
    Modify with change tracking
    Fetch repo + live API → parse change request → show diff → worktree branch → test → update JSON + docs + changelog.
  • /remove-dashboard
    Delete safely
    List dependent alert rules/doc links → confirm impact → remove from repo + live Grafana → archive backup.
  • /audit-dashboard
    Validate config and best practices
    JSON schema, PromQL/LogQL validity, alert thresholds, datasource connectivity, drift vs. repo version.
  • /list-dashboards
    Inventory with health status
    Scans all known dashboard locations; filter by application, status, category, or location.
Decision Point · project structure detection (priority order)
1
Ansible roles — roles/grafana/files/dashboards/ exists → ask category → store under <category>/<name>.json, doc as .README.md
2
Project-root grafana/ — grafana/dashboards/ exists, no Ansible → store there, doc in grafana/docs/
3
Docker Compose — docker-compose.yml exists, no roles/ → store in monitoring/grafana/dashboards/, doc in docs/dashboards/
4
Fallback — none match → create grafana/dashboards/ + grafana/docs/ automatically
User override
Explicit --path flag always wins; detected location is always shown before proceeding.
