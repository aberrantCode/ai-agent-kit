opbta-service

cross-repo dispatcher for internal services on svc.opbta.com

A thin dispatcher, invokable from any repo — not just AC_OPBTA. It resolves which repo invoked it, locates the AC_OPBTA repo, and then reads and follows that repo's own add-service / update-service / retire-service command files as the source of truth. No lifecycle logic is duplicated here. Metadata SoT lives in AC_OPBTA/inventory/services.yml; source repos carry only an .opbta-service breadcrumb (slug only) — zero metadata leakage.

The Six Commands

verb → repo command
/opbta-create
New deployment end-to-end. Reroutes to /opbta-update automatically if the slug already exists.
/opbta-update
Bump or reconcile an existing service. Reroutes to /opbta-create if the slug is absent.
/opbta-retire
Removes a service end-to-end — deprovisions and drops it from the catalog.
/opbta-inspect
Deep read on one service — config, health, and history in detail.
/opbta-manage
Day-2 ops: restart/stop/start, re-expose, rotate a secret, rename/move/resize.
/opbta-list
Catalog-wide read — health and drift summary across every service.
