continue-new-session-prompt — skill diagram

Continue in a New Session

Turns the open decisions from a running session into a prompt a fresh, unattended Claude Code session can execute — then launches that session in its own Windows Terminal tab, working in its own isolated worktree.

“Most of your effort belongs in capturing state, not in describing tasks.”

01 Entry Modes

Two doors into the same pipeline — they differ only in where the task scope comes from. Everything from Step 2 onward is identical.

Continue mode · default
/continue-new-session-prompt

Scope is recovered from this conversation — the numbered decision list in the most recent response. Ambiguous items with no safe default get settled now via AskUserQuestion, never passed downstream.

Scope mode
/dispatch-session-prompt <task>

Scope arrives with the invocation instead. Skip recovering a decision list — there may not be one. Parse the supplied task into the same ordered, concrete-instruction shape and proceed identically from Step 2.

03 The Seven Steps

Same sequence regardless of entry mode. Sequence matters more than completeness — a task that regenerates data invalidates prior reviews; ordering rationale gets written into the prompt itself.

1
Recover the items

Continue mode: resolve each numbered decision-list item into a concrete instruction using the recommendation already given. Scope mode: parse the supplied task into the same shape. Either way, drop questions about the handoff itself and settle them here — an unattended session cannot ask.

2
Order tasks by dependency

Walk every pair: does doing B before A waste A? The ordering rationale is written into the prompt so the new session holds the line when a later task looks tempting to do first.

3
Create the worktree

new-task-worktree.ps1 fetches, fast-forwards the local base to origin/<base>, and branches the worktree from the remote ref — never the possibly-stale local one. Optional -SyncDeps. Fast-forward only: a diverged local base stops the script rather than getting reset.

4
Write the prompt

Goes to docs/PROMPTS/<date>-<slug>.md in the main checkout — never inside the worktree, or it becomes part of the branch's diff. Covers orientation, hard constraints first and unmissable, state already established, ordered tasks, finishing instructions, haiku-delegation guidance, delegation logging, and /loop guidance for anything that needs polling.

Hard constraints matter most here — with --dangerously-skip-permissions on, this section is the only guardrail left.
5
Launch it

launch-claude-session.ps1 reads the prompt from a file — never passed inline through shell quoting — opens a new Windows Terminal tab pointed at the worktree, and runs with --dangerously-skip-permissions by default. The caller pins the model with -SetEnv "ANTHROPIC_MODEL=…", weighing the hardest task in the list, not the average.

6
Verify the launch and report

Confirm from the printed claude PID, not from Start-Process returning — a tab that opens and immediately dies looks identical to success otherwise. Report: worktree path, branch, prompt path, PID, model pinned and why, task order.

7
Retire the sessiononly once it has earned it

complete-task-session.ps1 refuses to delete anything unless all four gates hold:

  • branch is an ancestor of origin/<base> — actually merged
  • local base matches origin/<base> exactly — no unpushed/unpulled commits
  • repo tests pass — auto-detected, or -TestCommand, or an explained -SkipTests
  • -Confirmed is present — only after a human actually looked at the work

On success it deletes the prompt, removes the worktree, and deletes the local branch. Any failure leaves everything intact and recoverable.

04 Supporting Scripts

Five scripts do the mechanical work so the prompt only has to describe judgment.

new-task-worktree.ps1

Fetches, fast-forwards the local base to origin/<base>, branches the worktree from the remote ref, optionally syncs dependencies.

launch-claude-session.ps1

Reads the prompt from a file to avoid shell-quoting hazards, opens a new Windows Terminal tab pointed at the worktree, runs skip-permissions by default, lets the caller pin ANTHROPIC_MODEL.

log-delegation-outcome.ps1

Records each subagent-delegation outcome — escalation, redo, or drop — with a category, a failure mode, and one line of concrete evidence; also logs end-of-session totals as the ledger's denominator.

delegation-report.ps1

Reports failure share per category and model, flags routing pairs worth changing, and withholds recommendations below a minimum sample size.

complete-task-session.ps1

The four-gate retirement check — merged ancestor, base synced, tests pass, human -Confirmed — before deleting the prompt file, the worktree, and the branch.

05 If Windows Terminal Is Unavailable
Fallback path

wt ships with Windows 11 but can be absent on stripped-down or server installs. The launch script falls back to a plain pwsh console window automatically. If even that fails, the failure is never silent: report the prompt file path and the exact command the user can run themselves — the prompt, the expensive part, is already written and still useful.
