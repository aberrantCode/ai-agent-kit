Invariants · memorise, repo-wide
  • control
    Env-changing Ansible runs from ac-devops
    ssh ubuntu@192.168.30.15 → ~/repos/AC_OPBTA. Windows local runner is bootstrap / syntax-check / dry-run only.
  • secrets
    SOPS age key lives on one host only
    ac-devops ~/.config/sops/age/keys.txt. Load with community.sops.load_vars (name: X_creds) — never vars_files:.
  • firewall
    ac-firewall connection is local
    ansible_connection: local, ansible_become: false for the OPNsense host.
  • git
    Feature branch → PR → dev → main
    Release PR is dev → main. Never push directly to dev or main.
  • scripts
    Every action is a committed script
    scripts/*.sh, idempotent, set -euo pipefail — no ad hoc one-off commands.

Top Gotchas Across Tools
  • OPNsense rules need opt* vars, never UI descriptions (ISSUE-008).
  • vars_files: does not decrypt SOPS — use community.sops.load_vars (ISSUE-004).
  • community.general.yaml callback removed in ansible-core 2.20 — use result_format = yaml (ISSUE-006).
  • OPNsense module needs ansible_become: false (ISSUE-013) and api_port not port (ISSUE-007).
  • Never run sops from the workstation — the age key isn't there; always shell to ac-devops.

Mutating Commands · scaffold → PR → apply
  • /new-deployment
    Deploy a new service
    Scaffolds catalog entry, role wiring, compose fragment, DNS, Traefik route; applies via ac-devops.
  • /retire-service
    Inverse of /new-deployment
    Removes catalog entry, role wiring, compose fragment, Cloudflare + Unbound DNS, Traefik route, optionally OIDC + secrets + dashboard tile. Backs up to docs/audits/<date>-retired-<name>/ first; takes the ac-devops lock.
  • /new-firewall-rule
    Add an OPNsense rule
    Uses the opt* interface map above, not UI descriptions.
  • /rotate-secret
    Rotate a non-auth key
    Generated 48-hex value or --value; commits to current branch, syncs KeePass. Refuses auth-catalog keys and refuses to create new keys.
  • /add-host
    Provision a VM / LXC end-to-end
    AskUserQuestion interview, per-guest .env, runs provision-{vm,lxc}.sh, patches inventory, ansible -m ping bootstrap, opens PR. Refuses on hostname/IP/VMID/VLAN/OS-tag collisions.

Read-Only · helper script, no PR
/get-secret
Single key from any secrets/*.enc.yml — non-auth PSKs, webhooks, tokens.
/get-service-auth
URL + admin user + password, catalog-driven.
/list-service-auth
Every catalog service with source file + key paths.
/sops-status
Diagnoses SSH reach, age key, sops binary, KeePass mirror.
/check-hosts [scope]
Probes REACH / AUTH / SERVICE per host or group; auto-routes through ac-devops when ansible isn't on PATH.
Credential-Mutating · sops-secrets skill
/update-auth
Sets username/password via sops --set on ac-devops, commits, syncs KeePass, auto-runs the consuming playbook.
/rotate-service-password
Same, but generates the password first via openssl rand -hex 24.
/sync-keepass
Re-mirrors SOPS → KeePass on demand.
