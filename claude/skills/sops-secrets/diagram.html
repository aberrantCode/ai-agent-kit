SOPS Secrets Skill

operator domain knowledge · secrets/*.enc.yml · ac-devops-only decryption

The One Rule
Never run sops on the workstation. Always run it on ac-devops via SSH. The age key (~/.config/sops/age/keys.txt) lives on ac-devops only. If a script in this skill ever tries sops locally and the user waits for a timeout, that's a bug in the skill — fix the script, not the workflow.
what breaks when the rule is broken
Failure Modes · wrong move → what happens
  • get-secret.sh
    Run from Windows
    Exit 4 — "age key not found". Wasted round trip; you SSH anyway.
  • sops -d
    Run from Windows
    failed to MAC or "no key could decrypt".
  • local fallback
    command -v sops added to a script
    Some operator's stale local key half-works on some files, silently corrupts others.
  • playbook run
    Decrypt on a Proxmox/Docker host
    ISSUE-022 — sops binary not present; needs delegate_to: localhost.
Slash Commands · thin wrapper → committed script → ac-devops
/get-service-auth <service>
URL / username / password / token, three lines, from services-auth-catalog.yml.
/list-service-auth [filter]
Enumerate every catalog service with source file + key paths.
/sops-status
Probes ac-devops: SSH reach, age key, sops binary, parser, KeePass mirror.
/update-auth <service>
sops --set (atomic), commits + pushes, syncs KeePass, auto-applies the consuming playbook.
/rotate-service-password <service>
openssl rand -hex 24, then the /update-auth machinery.
/sync-keepass
Re-mirror SOPS → KeePass on demand after a manual ac-devops edit.
Not For
  • Creating a brand-new SOPS file — one-off, done by hand per secrets/README.md.
  • Rotating the age key itself — separate runbook.
  • Bulk-decrypting a whole file into chat — refused on principle; extract one key at a time.
  • New credential types — edit services-auth-catalog.yml first, then everything else flows.
vs /get-secret
/get-service-auth (this skill)
Higher-level, opinionated. Reach for it first for "I just need the admin login for X".
/get-secret <app> <key>
Older, lower-level. Kept as-is for raw SOPS access and data outside the catalog (PSKs, webhook URLs, internal tokens).
