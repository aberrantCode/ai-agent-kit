Default Architecture & Conventions

Structural constraints
Dependency Verification

Mandatory. Check package.json before importing any 3rd-party library. If missing, output the install command before the code — never assume a library exists.

Framework & Interactivity

React / Next.js, defaulting to Server Components. RSC safety: global state only in Client Components. Isolation: motion/glass components become isolated 'use client' leaves; servers render static layout only.

State Management

Local useState/useReducer for isolated UI. Global state reserved strictly for avoiding deep prop-drilling.

Styling Policy

Tailwind CSS for 90% of styling. Version lock: check package.json, never mix v4 syntax into a v3 project. T4 config guard: use @tailwindcss/postcss or the Vite plugin, never the old PostCSS plugin name.

Anti-Emoji Policy

Critical. No emojis anywhere — code, markup, text content, alt text. Replace with Radix/Phosphor icons or clean SVG primitives.

Responsiveness & Spacing

Standard breakpoints, max-w-[1400px] mx-auto. Viewport stability: min-h-[100dvh] instead of h-screen for heroes. Grid over flex-math: never calc(33%-1rem), always CSS Grid columns.

Icons. Import exclusively from @phosphor-icons/react or @radix-ui/react-icons (verify installed version); standardize one strokeWidth globally — either 1.5 or 2.0, never mixed.

Design Engineering Directives

6 bias-correction rules
Rule 1 · Deterministic Typography

Headlines default to text-4xl md:text-6xl tracking-tighter. Ban Inter for premium vibes — force Geist, Outfit, Cabinet Grotesk, or Satoshi. Serif is strictly banned on dashboards/software UI.

Rule 2 · Color Calibration

Max 1 accent color, saturation under 80%. The Lila Ban: no AI purple/blue glow aesthetic — neutral Zinc/Slate bases with one high-contrast accent. One palette for the entire output, no warm/cool gray drift.

Rule 3 · Layout Diversification

Anti-center bias. Centered hero/H1 is banned above LAYOUT_VARIANCE 4 — force split-screen 50/50, left/right content-asset splits, or asymmetric whitespace.

Rule 4 · Materiality & Anti-Card

Above VISUAL_DENSITY 7, generic card containers are banned — group with border-t, divide-y, or negative space. Elevation only when it communicates hierarchy; tint shadows to the background hue.

Rule 5 · Interactive UI States

Full interaction cycles are mandatory: skeletal loaders (no generic spinners), composed empty states, inline error reporting, and tactile :active feedback (-translate-y-[1px] / scale-[0.98]).

Rule 6 · Data & Form Patterns

Labels sit above inputs. Helper text optional but present in markup. Error text below input. Standard gap-2 per input block.

4

Creative Proactivity

Anti-slop implementation
Liquid Glass Refraction

Beyond backdrop-blur: add a 1px inner border (border-white/10) and inset shadow to simulate physical edge refraction.

Magnetic Micro-Physics

Triggers above MOTION_INTENSITY 5. Never useState for magnetic hover — exclusively Framer Motion's useMotionValue/useTransform outside the render cycle to prevent mobile performance collapse.

Perpetual Micro-Interactions

Above MOTION_INTENSITY 5: continuous infinite animation (pulse, typewriter, float, shimmer, carousel) on avatars/status dots/backgrounds, using spring physics (stiffness:100, damping:20), never linear easing.

Layout Transitions

Framer Motion's layout and layoutId for smooth re-ordering, resizing, and shared-element transitions across state changes.

Staggered Orchestration

staggerChildren (Framer) or CSS cascade delay for waterfall reveals. Critical: parent variants and children must share one Client Component tree — async data gets passed as props into a centralized parent motion wrapper.

Performance Guardrails

Non-negotiable
DOM Cost

Grain/noise filters go only on fixed, pointer-events-none pseudo-elements — never on scrolling containers, to avoid continuous GPU repaints.

Hardware Acceleration

Never animate top/left/width/height. Animate exclusively via transform and opacity.

Z-Index Restraint

No unprompted z-50/z-10 spam — reserved strictly for systemic layer contexts: sticky navbars, modals, overlays.

The Motion-Engine Bento Paradigm

SaaS dashboard architecture
Core Design Philosophy
  • Background #f9fafb, white cards, 1px slate-200/50 border
  • rounded-[2.5rem] containers with a wide diffusion shadow
  • Strict Geist / Satoshi / Cabinet Grotesk stack
  • Titles & descriptions sit outside and below cards
  • Generous p-8–p-10 interior padding
Animation Engine Specs
  • Spring physics only — stiffness:100, damping:20, never linear
  • layout/layoutId for shared-element transitions
  • Every card loops an "alive" active state infinitely
  • Performance critical: perpetual motion memoized and isolated in its own microscopic Client Component — never re-renders the parent

Final Pre-Flight Check

Last filter before output
  • Global state used to avoid deep prop-drilling, not arbitrarily
  • Mobile layout collapse guaranteed for high-variance designs (w-full, px-4, max-w-7xl mx-auto)
  • Full-height sections use min-h-[100dvh], never the bugged h-screen
  • useEffect animations contain strict cleanup functions
  • Empty, loading, and error states are all provided
  • Cards omitted in favor of spacing wherever possible
  • CPU-heavy perpetual animations strictly isolated in their own Client Components
